How to Practice Universal Design

Universal design happens through small choices. This page demonstrates useful ways to make websites work for more people.

Start with Solid Structure

Begin by writing clean, semantic HTML. Use proper headings, lists, and landmarks so both people and assistive technologies can understand the page.

  • Use a single <h1> for the page topic, then nested headings in order
  • Wrap the main content in <main> and supporting content in <aside>
  • Use lists, tables, and forms only when they match the meaning of the content
  • Add a skip link so keyboard users can jump straight to the main section

Design for Different Ways of Interacting

Everyone uses the web in unique ways. Some rely on keyboards, some screen readers, and many browse on small touch screens.

  • Make sure every interactive element can be reached and used with a keyboard alone
  • Use visible focus styles so it is clear where the keyboard is on the page
  • Give images meaningful alternative text, or empty alt text when they are purely decorative
  • Use clear labels, legends, and instructions for forms, instead of only placeholder text
  • Provide enough color contrast and avoid using color as the only way to show meaning

A Quick Checklist for this Site

This project uses several universal design techniques you can reuse in other work.

  • Mobile-first, responsive layout using CSS grid so content reflows on different screen sizes
  • Semantic landmarks: header, main, asides, and footer to organize the page
  • Skip link to jump past repeated content
  • Descriptive alt text and captions for meaningful images and infographics
  • Accessible contact form with labels, a fieldset, and a legend in the home page footer
  • Consistent button styles and color themes that keep text readable against the background
